Sistema de Módulos por Tipo de Proyecto

La estrategia gobierna qué entra. Cada vertical tiene su flujo de módulos estándar. Las herramientas conectan el pipeline de información. El scope se cierra cuando el cliente valida software funcionando.

Directiva Marco · Obligatoria para todo proyecto

Estrategia del Proyecto

Toda iniciativa se filtra contra esta directiva antes de entrar al flujo. Vive en Notion.
Contexto
Cómo funciona el negocio del cliente, qué rol cumple el producto, estado actual y presiones.
Problema prioritario
Qué problema mover, para quién (ICP), por qué ahora.
Métrica norte
Una métrica principal + 2-3 métricas de soporte con baseline y target.
Apuestas / Hipótesis
Qué creemos que va a mover la aguja. Con criterios de validación.
Qué SÍ hacer
Acciones explícitas alineadas al foco del proyecto.
Qué NO hacer
Límites explícitos. Qué decidimos no priorizar ahora y por qué.
Matriz Go / No-Go
Criterios estándar para decidir si una propuesta avanza: impacto, viabilidad operativa, control de promesa. Cada criterio con definición de Go / Condicional / No-Go.

Validación Cliente → Roadmap · Feedback del cliente ajusta foco de próximas iniciativas.
Entregable Discovery → Delivery Puntual · Discovery validado puede convertirse en proyecto de delivery.
