1. ESTRATEGIA GENERAL MENÚ

Matriz Go / No-Go

Filtro estricto para decidir si una propuesta avanza o se descarta.

Criterios de Aprobación

  • Impacto directo en reducir reventa/bypass.
  • Viable en el ecosistema actual (Front/Gateway).
  • Boca tiene control del End-to-End.

Criterios de Rechazo

  • Cambios profundos en el Backend de Boca.
  • Features no relacionadas a seguridad.
  • Bloqueos automáticos de socios.

6. SISTEMA MENÚ

Feedback Loops

Cada ciclo cerrado alimenta la estrategia del siguiente.

Resultados → Estrategia

Revisión de métricas post-deploy. Confirma o invalida hipótesis y ajusta prioridades.

Soporte → Backlog

Patrones recurrentes detectados en el Triage fuerzan repriorización de deuda técnica.

Validación → Roadmap

Feedback del cliente durante el testing re-enfoca las iniciativas futuras.

Eventos → Aprendizaje

El uso real durante partidos de alta demanda estresa y valida el producto final.
